Los recursos de demostración son máquinas virtuales de Azure preconfiguradas que se despliegan mediante Terraform.
Cada componente es una VM independiente con software específico instalado y configurado
a través de cloud-init en el primer arranque.

## Selección de componentes

Los componentes son bloques de construcción. Selecciónelos según las necesidades de su demostración:

- **Servidor de origen** proporciona aplicaciones web vulnerables que sirven como objetivos para
  las funcionalidades de seguridad de F5 XC. La mayoría de las demostraciones lo necesitan.
- **Generador de tráfico** produce tráfico de ataque y tráfico legítimo contra su entorno
  de demostración. Necesario al demostrar la eficacia del WAF, la detección de bots o la
  protección de API.
- **Simulador de CDN** coloca un borde CDN basado en NGINX entre los clientes y el origen,
  inyectando encabezados específicos del proveedor. Utilizado para demostraciones de integración con CDN.

## Prerrequisitos compartidos

Todos los componentes requieren:

- Una suscripción de Azure con acceso de colaborador
- Terraform 1.5 o posterior
- Azure CLI autenticado (`az login`)
- Un par de claves SSH (ruta predeterminada `~/.ssh/id_ed25519.pub`)

## Patrón de despliegue

Cada componente sigue el mismo flujo de trabajo de Terraform:

1. Clone el repositorio del componente
2. `cd terraform/`
3. Copie `terraform.tfvars.example` a `terraform.tfvars` y establezca las variables requeridas (`subscription_id` es obligatorio para todos; algunos componentes tienen entradas obligatorias adicionales)
4. `terraform init && terraform plan && terraform apply`
5. Verifique mediante el endpoint de verificación de estado del componente

Su identificador de desplegador se resuelve automáticamente desde su cuenta de Azure AD y se incorpora en todos los nombres de recursos (por ejemplo, `rg-origin-server-lab-rmordasiewicz`).

## Integración entre componentes

Los componentes se combinan para crear entornos de demostración completos:

```
Traffic Generator ──► F5 XC HTTP LB (WAF/Bot/API/CSD) ──► Origin Server
                                     ▲
                               CDN Simulator
```

**Orden de despliegue:**

1. **Servidor de origen** — despliegue primero, aloja las aplicaciones que todos los demás componentes tienen como objetivo
2. **Balanceador de carga HTTP de F5 XC** — configure un origin pool apuntando a la IP pública del servidor de origen, luego cree un balanceador de carga HTTP con políticas de WAF, Bot Defense, API Security o Client-Side Defense
3. **Generador de tráfico** y/o **Simulador de CDN** — despliegue en último lugar, apuntando al FQDN del balanceador de carga de F5 XC

Las variables de Terraform de cada componente hacen referencia a las salidas de los componentes desplegados previamente. Consulte la documentación del componente para conocer los requisitos específicos de variables.

Utilice la [calculadora de precios de Azure](https://azure.microsoft.com/en-us/pricing/calculator/) para obtener estimaciones de costos actuales. Ejecute `terraform destroy` cuando el laboratorio esté inactivo para detener los cargos.
