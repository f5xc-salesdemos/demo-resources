डेमो संसाधन पूर्व-कॉन्फ़िगर Azure वर्चुअल मशीनें हैं जिन्हें Terraform के माध्यम से डिप्लॉय किया जा सकता है।
प्रत्येक कंपोनेंट एक स्वतंत्र VM है जिसमें उद्देश्य-विशिष्ट सॉफ़्टवेयर इंस्टॉल किया गया है और
पहले बूट पर cloud-init के माध्यम से कॉन्फ़िगर किया गया है।

## कंपोनेंट का चयन

कंपोनेंट बिल्डिंग ब्लॉक हैं। अपने डेमो की आवश्यकता के अनुसार इन्हें चुनें:

- **ऑरिजिन सर्वर** कमजोर वेब एप्लिकेशन प्रदान करता है जो F5 XC सुरक्षा सुविधाओं के लक्ष्य के रूप में काम करते हैं। अधिकांश डेमो के लिए इसकी आवश्यकता होती है।
- **ट्रैफ़िक जनरेटर** आपके डेमो वातावरण के विरुद्ध अटैक और वैध ट्रैफ़िक उत्पन्न करता है। WAF प्रभावकारिता, बॉट डिटेक्शन, या API सुरक्षा प्रदर्शित करते समय इसकी आवश्यकता होती है।
- **CDN सिम्युलेटर** क्लाइंट और ऑरिजिन के बीच एक NGINX-आधारित CDN एज रखता है, जो वेंडर-विशिष्ट हेडर इंजेक्ट करता है। CDN इंटीग्रेशन डेमो के लिए उपयोग किया जाता है।

## साझा पूर्वापेक्षाएँ

सभी कंपोनेंट के लिए आवश्यक है:

- कंट्रीब्यूटर एक्सेस के साथ एक Azure सब्सक्रिप्शन
- Terraform 1.5 या उससे बाद का संस्करण
- Azure CLI प्रमाणित (`az login`)
- एक SSH की पेयर (डिफ़ॉल्ट पथ `~/.ssh/id_ed25519.pub`)

## डिप्लॉयमेंट पैटर्न

प्रत्येक कंपोनेंट समान Terraform वर्कफ़्लो का पालन करता है:

1. कंपोनेंट रिपो क्लोन करें
2. `cd terraform/`
3. `terraform.tfvars.example` को `terraform.tfvars` में कॉपी करें और आवश्यक वेरिएबल सेट करें (सभी के लिए `subscription_id` आवश्यक है; कुछ कंपोनेंट में अतिरिक्त आवश्यक इनपुट होते हैं)
4. `terraform init && terraform plan && terraform apply`
5. कंपोनेंट के हेल्थ चेक एंडपॉइंट के माध्यम से सत्यापित करें

आपका डिप्लॉयर पहचानकर्ता आपके Azure AD खाते से स्वतः रिज़ॉल्व होता है और सभी संसाधन नामों में एम्बेड किया जाता है (जैसे, `rg-origin-server-lab-rmordasiewicz`)।

## क्रॉस-कंपोनेंट इंटीग्रेशन

कंपोनेंट मिलकर संपूर्ण डेमो वातावरण बनाते हैं:

```
Traffic Generator ──► F5 XC HTTP LB (WAF/Bot/API/CSD) ──► Origin Server
                                     ▲
                               CDN Simulator
```

**डिप्लॉयमेंट क्रम:**

1. **ऑरिजिन सर्वर** — पहले डिप्लॉय करें, यह उन एप्लिकेशन को होस्ट करता है जिन्हें अन्य सभी कंपोनेंट लक्षित करते हैं
2. **F5 XC HTTP लोड बैलेंसर** — ऑरिजिन सर्वर के पब्लिक IP की ओर इंगित करते हुए एक ऑरिजिन पूल कॉन्फ़िगर करें, फिर WAF, Bot Defense, API Security, या Client-Side Defense पॉलिसी के साथ एक HTTP लोड बैलेंसर बनाएं
3. **ट्रैफ़िक जनरेटर** और/या **CDN सिम्युलेटर** — अंत में डिप्लॉय करें, F5 XC लोड बैलेंसर FQDN की ओर इंगित करते हुए

प्रत्येक कंपोनेंट के terraform वेरिएबल पहले डिप्लॉय किए गए कंपोनेंट के आउटपुट को संदर्भित करते हैं। विशिष्ट वेरिएबल आवश्यकताओं के लिए कंपोनेंट दस्तावेज़ देखें।

वर्तमान लागत अनुमानों के लिए [Azure मूल्य निर्धारण कैलकुलेटर](https://azure.microsoft.com/en-us/pricing/calculator/) का उपयोग करें। शुल्क रोकने के लिए लैब निष्क्रिय होने पर `terraform destroy` चलाएं।
